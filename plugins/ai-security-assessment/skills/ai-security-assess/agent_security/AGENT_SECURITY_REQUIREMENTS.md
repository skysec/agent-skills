### T1 — Memory Poisoning

| Field | Content |
|-------|---------|
| **TID** | T1 |
| **Threat Name** | Memory Poisoning |
| **Description** | Memory Poisoning involves exploiting an AI's memory systems — both short and long-term — to introduce malicious or false data and exploit the agent's context. This can lead to altered decision-making and unauthorized operations. Attacks include direct prompt injections into isolated memory and exploitation of shared memory structures affecting multiple agents and users across sessions. |
| **Mitigations** | Implement memory content validation and automated scanning for anomalous candidate memory insertions. Enforce session isolation so agents do not carry unintended knowledge across user sessions. Apply context-aware memory access policies limiting retrieval to current operational scope. Require source attribution for memory updates. Implement pre-commit multi-agent validation before persisting cross-session memory changes. Use probabilistic truth-checking against trusted sources before committing to long-term storage. Deploy anomaly detection on unexpected memory log updates. Use rollback mechanisms and forensic memory snapshots to restore validated states. |
| **MITRE ATLAS Mapping** | **PRIMARY** ★ Memory Manipulation — Persistence (AML.TA0006): Directly models LLM long-term memory alteration to persist malicious changes across future sessions. ★ AI Agent Context Poisoning — Persistence (AML.TA0006): Persistent manipulation of agent context window to influence behavior across interactions. ★ Thread Injection — Execution (AML.TA0005): Injects malicious instructions into chat threads affecting short-term memory for session duration. **SECONDARY** AML.T0051 LLM Prompt Injection — Execution (AML.TA0005): Common delivery vector for injecting poisoned content into agent memory. AML.T0020 Poison Training Data — ML Attack Staging (AML.TA0001): Analogous poisoning technique at the training layer; relevant when memory stores feed retraining pipelines. AML.T0018 Backdoor ML Model — Persistence (AML.TA0006): Where poisoned memory triggers backdoor behavior across sessions. |
| **ATLAS Coverage** | ✅ Strong — Oct 2025 agentic additions directly model this threat. Pre-2025 ATLAS lacked persistent agent memory as a distinct attack surface. |

### T2 — Tool Misuse

| Field | Content |
|-------|---------|
| **TID** | T2 |
| **Threat Name** | Tool Misuse |
| **Description** | Tool Misuse occurs when attackers manipulate AI agents to abuse their integrated tools through deceptive prompts or commands, operating within authorized permissions. This includes Agent Hijacking, where an AI agent ingests adversarially manipulated data and subsequently executes unintended actions, potentially triggering malicious tool interactions. Unlike traditional exploits, this attack leverages AI's ability to chain tools and execute complex sequences of seemingly legitimate actions, making detection difficult. |
| **Mitigations** | Enforce strict tool access verification and pre-execution validation. Implement tool rate-limiting and monitor tool usage patterns for anomalies. Validate agent instructions and set clear operational boundaries. Maintain execution logs tracking all AI tool calls for forensic auditing. Require explicit user approval for tool executions involving financial, medical, or administrative functions. Detect abnormal tool execution frequency and unexpected side effects. Apply just-in-time (JIT) tool access, revoking permissions immediately after use. Sandbox tool execution in isolated containerized environments. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0053 LLM Plugin Compromise — Execution (AML.TA0005): Directly models adversarial exploitation of AI plugin/tool integrations. ★ Tool Definitions Discovery — Discovery (AML.TA0008): Adversary reconnaissance of available agent tools to plan misuse attack chain. ★ Exfiltration via AI Agent Tool Invocation — Exfiltration (AML.TA0010): Directly models using agent write-capable tools (email, CRM, document creation) to exfiltrate data. **SECONDARY** AML.T0051 LLM Prompt Injection — Execution (AML.TA0005): Primary delivery vector for hijacking tool invocation logic. AML.T0050 Command and Scripting Interpreter — Execution (AML.TA0005): Agent-generated scripts or shell commands as a tool misuse vector. AML.T0011 User Execution — Execution (AML.TA0005): Execution triggered by agent decision-making rather than direct user action. ★★ AI Agent Clickbait (AML.T0100) — Execution (AML.TA0005): Web/document content crafted to manipulate agent tool invocation decisions. ★ Activation Triggers — Discovery (AML.TA0008): Identifying trigger conditions to time tool misuse attacks. |
| **ATLAS Coverage** | ✅ Strong — both pre-existing and Oct 2025 techniques cover this well. Exfiltration via AI Agent Tool Invocation is a precise agentic match. |

### T3 — Privilege Compromise

| Field | Content |
|-------|---------|
| **TID** | T3 |
| **Threat Name** | Privilege Compromise |
| **Description** | Privilege Compromise arises when attackers exploit weaknesses in permission management to perform unauthorized actions. This often involves dynamic role inheritance or misconfigurations. In agentic AI, agents can autonomously inherit permissions, creating security blind spots where temporary or inherited privileges are abused. Agents can chain multiple tools in unexpected ways to bypass intended security controls and implicit privilege escalation can occur when agents inherit excessive permissions from user sessions or service tokens. |
| **Mitigations** | Implement granular permission controls and dynamic access validation. Monitor role changes robustly and audit elevated privilege operations thoroughly. Prevent cross-agent privilege delegation unless explicitly authorized through predefined workflows. Apply just-in-time access for AI tools, revoking permissions immediately after use. Require two-agent or human validation for high-risk AI actions involving authentication changes. Detect and flag role inheritance anomalies in real time. Apply time-based restrictions on privilege elevation. Enforce mutual authentication for AI-to-AI interactions. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0012 Valid Accounts — Initial Access (AML.TA0004): Exploitation of legitimately provisioned agent accounts and inherited session privileges. AML.T0055 Unsecured Credentials — Credential Access (AML.TA0013): Exploitation of improperly secured agent API keys, service tokens, and session credentials. ★ Credentials from AI Agent Configuration — Credential Access (AML.TA0013): Directly models extraction of API keys and passwords from agent configuration, enabling privilege escalation. **SECONDARY** AML.T0054 LLM Jailbreak — Defense Evasion (AML.TA0007): Bypassing authorization guardrails embedded in agent system prompts to access restricted functionality. ★ Discover AI Agent Configuration — Discovery (AML.TA0008): Reconnaissance of agent configuration to identify over-provisioned permissions and exploitable scopes. ★ Modify AI Agent Configuration — Persistence (AML.TA0006): Altering agent configuration to retain or expand elevated privileges persistently. ★★ AI Service API — Execution (AML.TA0005): Living-off-the-land privilege escalation via legitimate AI service API infrastructure. |
| **ATLAS Coverage** | ✅ Good — Valid Accounts and Unsecured Credentials are strong pre-existing mappings. Oct 2025 additions add precision for configuration-based credential theft. |

### T4 — Resource Overload

| Field | Content |
|-------|---------|
| **TID** | T4 |
| **Threat Name** | Resource Overload |
| **Description** | Resource Overload targets the computational, memory, and service capacities of AI systems to degrade performance or cause failures. Unlike traditional DoS attacks, AI agents are especially vulnerable because they autonomously schedule, queue, and execute tasks across sessions without human oversight, can self-trigger tasks, spawn additional processes, and coordinate with multiple agents — leading to exponential resource consumption. |
| **Mitigations** | Deploy resource management controls and implement adaptive scaling mechanisms. Establish per-agent quotas and monitor system load in real time. Implement AI rate-limiting policies restricting high-frequency task requests per agent session. Track cumulative resource consumption across multiple AI agents to prevent collective overload. Limit concurrent AI-initiated system modification requests. Enforce auto-suspension of AI processes exceeding predefined resource consumption thresholds. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0029 Denial of ML Service — Impact (AML.TA0011): Directly models flooding or degrading AI service availability. AML.T0034 Cost Harvesting — Impact (AML.TA0011): Adversaries exploit AI resource intensity by crafting computationally expensive inputs to inflate costs and degrade performance. AML.T0046 Spamming ML System with Chaff Data — Impact (AML.TA0011): Flooding AI detection systems with synthetic false positives to overwhelm processing capacity. **SECONDARY** AML.T0043 Craft Adversarial Data — ML Attack Staging (AML.TA0001): Creating specially crafted inputs (e.g., "sponge" examples) that activate excessive computation in neural networks. AML.T0048 External Harms — Impact (AML.TA0011): Operational disruption as an intended adversarial outcome. |
| **ATLAS Coverage** | ✅ Strong — three direct ATLAS techniques model this well. AML.T0029 (DoS) and AML.T0034 (Cost Harvesting) are precise pre-existing matches. Agentic self-triggering aspect is not yet explicitly modeled in ATLAS. |

### T5 — Cascading Hallucination Attacks

| Field | Content |
|-------|---------|
| **TID** | T5 |
| **Threat Name** | Cascading Hallucination Attacks |
| **Description** | These attacks exploit an AI's tendency to generate contextually plausible but false information, which propagates through agent memory, tool use, or multi-agent interactions, amplifying misinformation across decision-making steps. In single-agent environments, hallucinations compound through self-reinforcement mechanisms such as reflection, self-critique, or memory recall. In multi-agent systems, misinformation propagates through inter-agent communication loops leading to cascading errors and systemic failures — particularly dangerous in healthcare, finance, and cybersecurity contexts. |
| **Mitigations** | Establish robust output validation mechanisms and implement behavioral constraints. Deploy multi-source validation and ensure ongoing system corrections through feedback loops. Require secondary validation of AI-generated knowledge before use in critical decision-making. Apply probabilistic truth-checking mechanisms against trusted sources before committing knowledge to long-term storage. Limit knowledge propagation from unverified sources. Track AI-generated knowledge lineage to enable forensic investigation of misinformation spread. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0062 Discover LLM Hallucinations — Discovery (AML.TA0008): Adversarial reconnaissance to identify hallucination-prone model behaviors for exploitation. AML.T0060 Publish Hallucinated Entities — Impact (AML.TA0011): Deliberately inducing LLMs to generate and publish false entities, spreading misinformation at scale. ★ AI Agent Context Poisoning — Persistence (AML.TA0006): Poisoning agent context to seed false information that cascades through subsequent reasoning loops. **SECONDARY** ★ Memory Manipulation — Persistence (AML.TA0006): Persisting hallucinated facts in agent long-term memory so they are reinforced in future sessions. AML.T0020 Poison Training Data — ML Attack Staging (AML.TA0001): Where hallucinations are seeded through poisoned training/fine-tuning data rather than runtime injection. ★ RAG Database Prompting — Collection (AML.TA0009): Extracting or triggering hallucinated responses via specially crafted RAG queries. |
| **ATLAS Coverage** | ⚠️ Partial — AML.T0060 and AML.T0062 cover hallucination discovery and publishing. The multi-agent self-reinforcement cascade dimension — hallucinations propagating through reflection loops and inter-agent communication — is not yet explicitly modeled in ATLAS. This is a coverage gap for the most dangerous agentic variant of this threat. |

### T6 — Intent Breaking & Goal Manipulation

| Field | Content |
|-------|---------|
| **TID** | T6 |
| **Threat Name** | Intent Breaking & Goal Manipulation |
| **Description** | This threat exploits vulnerabilities in an AI agent's planning and goal-setting capabilities, allowing attackers to manipulate or redirect the agent's objectives and reasoning. Attackers exploit the lack of separation between data and instructions in AI agents using prompt injections, compromised data sources, or malicious tools to alter planning, reasoning, and self-evaluation. This extends basic prompt injection risks — adversaries can inject adversarial objectives that shift an agent's long-term reasoning processes through gradual sub-goal injection, reflection loop traps, or meta-learning manipulation. |
| **Mitigations** | Implement planning validation frameworks and boundary management for reflection processes. Apply dynamic protection mechanisms for goal alignment. Deploy AI behavioral auditing using a secondary model to check the agent and flag significant goal deviations. Use goal consistency validation to detect and block unintended behavioral shifts. Track goal modification request frequency per AI agent. Apply behavioral constraints to prevent AI self-reinforcement loops from adjusting objectives beyond predefined operational parameters. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0051 LLM Prompt Injection — Execution (AML.TA0005): Core delivery mechanism for injecting adversarial objectives into agent planning contexts; includes both direct and indirect sub-techniques. ★ AI Agent Context Poisoning — Persistence (AML.TA0006): Directly models persistent manipulation of agent planning context to redirect long-term reasoning goals. ★ Thread Injection — Execution (AML.TA0005): Injecting malicious planning instructions into the agent's active reasoning thread. **SECONDARY** AML.T0054 LLM Jailbreak — Defense Evasion (AML.TA0007): Bypassing safety guardrails that constrain agent goal-setting to authorized objectives. AML.T0061 LLM Prompt Self-Replication — Persistence (AML.TA0006): Self-replicating prompts that persistently redirect agent goals across interactions. ★★ AI Agent Clickbait (AML.T0100) — Execution (AML.TA0005): Environmental content that covertly manipulates agent planning decisions. ★ Activation Triggers — Discovery (AML.TA0008): Identifying trigger conditions that initiate goal-manipulation sequences automatically. |
| **ATLAS Coverage** | ✅ Strong — AML.T0051 is the core technique, with Oct 2025 additions providing precise modeling of persistent, context-level goal manipulation in agentic systems. |

### T7 — Misaligned & Deceptive Behaviors

| Field | Content |
|-------|---------|
| **TID** | T7 |
| **Threat Name** | Misaligned & Deceptive Behaviors |
| **Description** | AI agents may execute harmful or disallowed actions by exploiting deceptive reasoning or misinterpreting goals to fulfill their objectives. This threat arises when autonomous agents develop misaligned strategies without direct malicious input, leading to unintended or catastrophic outcomes. It is distinct from hallucinations — the behavior emerges from advanced reasoning capabilities rather than random error or prompt failure. Scenarios include bypassing ethical constraints to achieve objectives, self-preservation behaviors, AI deception for task completion (e.g., hiring a human to solve a CAPTCHA), and goal-driven lethal decision-making. |
| **Mitigations** | Train models to recognize and refuse harmful tasks. Enforce policy restrictions and require human confirmations for high-risk actions. Implement logging and monitoring of agent reasoning chains. Utilize deception detection strategies: behavioral consistency analysis, truthfulness verification models, and adversarial red teaming to assess inconsistencies between AI outputs and expected reasoning pathways. Restrict tool access to minimize the attack surface for autonomous action. Implement validation mechanisms to detect and filter manipulated responses using guardrails and moderation APIs. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0054 LLM Jailbreak — Defense Evasion (AML.TA0007): Adversarially circumventing safety constraints and guardrails to enable disallowed autonomous actions. AML.T0015 Evade ML Model — Defense Evasion (AML.TA0007): Agent behavior designed to evade detection by monitoring systems while pursuing misaligned objectives. **SECONDARY** AML.T0051 LLM Prompt Injection — Execution (AML.TA0005): Used to initiate the conditions under which deceptive reasoning patterns emerge. AML.T0048 External Harms — Impact (AML.TA0011): Outcome classification when misaligned behaviors produce real-world negative consequences. AML.T0043 Craft Adversarial Data — ML Attack Staging (AML.TA0001): Creating inputs specifically designed to trigger emergent deceptive reasoning states. ★ AI Agent Context Poisoning — Persistence (AML.TA0006): Persistent context manipulation that primes agents for ongoing misaligned behavior. |
| **ATLAS Coverage** | ⚠️ Partial — Jailbreaking and evasion techniques provide some coverage. However, the emergent strategic deception dimension — agents autonomously developing and executing deceptive strategies without direct prompting — is not yet explicitly modeled in ATLAS. This remains a frontier area in adversarial AI research. |

### T8 — Repudiation & Untraceability

| Field | Content |
|-------|---------|
| **TID** | T8 |
| **Threat Name** | Repudiation & Untraceability |
| **Description** | Occurs when actions performed by AI agents cannot be traced back or accounted for due to insufficient logging or transparency in decision-making processes. This risk is exacerbated by opaque decision-making, lack of action tracking, and challenges in reconstructing agent behaviors across multiple — often parallel — reasoning and execution pathways in agentic AI. This creates compliance violations, security gaps, and operational blind spots in high-stakes environments including finance, healthcare, and cybersecurity. |
| **Mitigations** | Implement comprehensive logging with cryptographic verification, enriched metadata, and real-time monitoring to ensure accountability and traceability. Require AI-generated logs to be cryptographically signed and immutable for regulatory compliance. Enforce cryptographic logging and immutable audit trails to prevent log tampering. Implement real-time anomaly detection on AI decision-making workflows. Monitor and log human overrides of AI recommendations. Detect and flag decision reversals in high-risk workflows. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0015 Evade ML Model — Defense Evasion (AML.TA0007): Agents or adversaries manipulating agent behavior to minimize detectable artifacts in logs and monitoring systems. ★ Modify AI Agent Configuration — Persistence (AML.TA0006): Configuration changes that disable, reduce, or falsify agent logging and audit trail generation. **SECONDARY** AML.T0048 External Harms — Impact (AML.TA0011): Harm that goes undetected or unattributed due to inadequate traceability. AML.T0050 Command and Scripting Interpreter — Execution (AML.TA0005): Script execution paths that bypass standard application-layer logging. AML.T0025 Exfiltration via Cyber Means — Exfiltration (AML.TA0010): Data or model exfiltration that leaves no traceable record due to log gaps. |
| **ATLAS Coverage** | ⚠️ Partial — No ATLAS technique directly models log suppression or audit trail manipulation specific to AI agent decision pathways. The Defense Evasion tactic (AML.TA0007) is the closest home. The specific challenge of parallel agentic reasoning producing untraceable distributed actions is an ATLAS gap. |

### T9 — Identity Spoofing & Impersonation / Agent Identity Compromise

AML.T0052 Phishing — Initial Access (AML.TA0004): Social engineering to obtain agent credentials or hijack user sessions that agents operate under. **SECONDARY** AML.T0021 Establish Accounts — Resource Development (AML.TA0003): Creating fraudulent agent identities or shadow accounts that inherit legitimate agent privileges. ★ Modify AI Agent Configuration — Persistence (AML.TA0006): Configuration modification to redirect agent identity assertions or trust relationships. ★ RAG Credential Harvesting — Credential Access (AML.TA0013): Extracting credentials inadvertently stored in RAG datastores the agent uses. |
| **ATLAS Coverage** | ✅ Strong — Valid Accounts and Unsecured Credentials provide solid pre-existing coverage. The Oct 2025 addition of Credentials from AI Agent Configuration precisely models the persistent agent identity takeover scenario (Entra Agent ID). |

Employ dynamic intervention thresholds to adjust the level of human oversight based on risk, confidence, and context. Apply adaptive workload distribution across human reviewers to prevent decision fatigue for individuals. |

AML.T0034 Cost Harvesting — Impact (AML.TA0011): Forcing costly high-frequency human review cycles as part of an operational disruption strategy. **SECONDARY** AML.T0048 External Harms — Impact (AML.TA0011): Realized harm resulting from degraded human oversight quality. AML.T0043 Craft Adversarial Data — ML Attack Staging (AML.TA0001): Crafting inputs specifically designed to appear high-risk and trigger maximum HITL intervention. |
| **ATLAS Coverage** | ⚠️ Partial — Resource exhaustion and spam techniques provide partial coverage. However, the specific adversarial exploitation of human cognitive limitations in AI oversight workflows — decision fatigue engineering, trust mechanism subversion, and HITL interface manipulation — is not explicitly modeled in ATLAS. This is a gap reflecting the human-factors dimension of agentic security. |

### T16 — Insecure Inter-Agent Protocol Abuse

| Field | Content |
|-------|---------|
| **TID** | T16 |
| **Threat Name** | Insecure Inter-Agent Protocol Abuse |
| **Description** | Attacks target flaws in protocols like MCP (Model Context Protocol) or A2A (Agent-to-Agent), such as consent bypass or context hijacking, leading to unauthorized agent actions. This involves abusing the commands and trust embedded in these protocols — manipulating server responses, injecting context, or misleading agents through poorly defined tool descriptions or ambiguous consent flows. When protocol specifications are loosely enforced or implementations lack input validation and strong identity binding, attackers can hijack agent behavior, escalate privileges, or bypass guardrails entirely. |
| **Mitigations** | Enforce strong authentication between agents. Sanitize and validate all protocol-level data including context payloads and tool metadata to prevent injection or misinterpretation. Restrict agent-to-agent delegation to tightly scoped functions. Log all inter-agent communications and tool invocations to detect anomalies and support post-incident analysis. Encrypt communications to prevent adversary-in-the-middle attacks. Detect and flag cascading or recursive tool execution patterns triggered across agents via protocol misuse. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0053 LLM Plugin Compromise — Execution (AML.TA0005): Directly models exploitation of plugin/tool integrations — MCP servers are the contemporary form of LLM plugins. ★ AI Agent Context Poisoning — Persistence (AML.TA0006): Context hijacking via malicious MCP/A2A server responses, injecting poisoned context payloads interpreted as trusted protocol context. ★ Tool Definitions Discovery — Discovery (AML.TA0008): Probing MCP tool registries to identify exploitable tool descriptions and authorization boundaries. **SECONDARY** AML.T0012 Valid Accounts — Initial Access (AML.TA0004): Exploiting legitimate agent identity credentials within protocol trust chains. AML.T0051 LLM Prompt Injection — Execution (AML.TA0005): Embedding prompt injection payloads within protocol-level data (tool descriptions, context payloads, server responses). ★ Discover AI Agent Configuration — Discovery (AML.TA0008): Mapping MCP server registrations and A2A agent capabilities to identify attack surface. ★ Embedded Knowledge Discovery — Discovery (AML.TA0008): Discovering data sources exposed to agents via MCP server registrations. ★★ AI Service API — Execution / Collection (AML.TA0005 / AML.TA0009): Exploiting AI service API infrastructure as a protocol-level attack surface for living-off-the-land operations within agent ecosystems. |

### T17 — Supply Chain Compromise

| Field | Content |
|-------|---------|
| **TID** | T17 |
| **Threat Name** | Supply Chain Compromise |
| **Description** | A compromised supply chain can result in vulnerable, malicious, outdated, or otherwise harmful components being included in the agent, allowing an attacker to manipulate agent actions, obtain data, or run arbitrary code. This can occur via models, libraries, tools, poisoned build environments, or other system components. Unlike standalone LLM deployments, agents are long-lived stateful entities capable of persistently executing actions or propagating logic — making a single supply chain compromise capable of triggering privilege escalation, behavioral drift, or systemic misuse across interlinked agents. |
| **Mitigations** | Secure agent ecosystems by digitally signing artifacts. Use verifiable SBOMs (AIBOMs, Agent SBOMs) and apply version control with peer review. Enforce strong authentication across the supply chain. Restrict untrusted tool installations and run agents in sandboxed isolated environments. Continuously monitor supply chain dependencies for security threats and indicators of compromise. Red-team agents with simulated supply chain attacks to validate defenses. Apply version control and peer review for prompt/script repositories and memory definitions, just as for code. |
| **MITRE ATLAS Mapping** | **PRIMARY** AML.T0010 ML Supply Chain Compromise — Initial Access (AML.TA0004): The parent technique directly modeling compromise of any ML/AI supply chain component. AML.T0010.002 ML Supply Chain: Data — Initial Access (AML.TA0004): Poisoning training or fine-tuning datasets incorporated via the supply chain. AML.T0010.003 ML Supply Chain: Model — Initial Access (AML.TA0004): Distributing compromised pre-trained or fine-tuned models (e.g., via open-source model hubs). AML.T0058 Publish Poisoned Models — Resource Development (AML.TA0003): Distributing backdoored models that agents adopt through model supply chains. **SECONDARY** ★ Modify AI Agent Configuration — Persistence (AML.TA0006): Supply chain compromise that modifies agent configuration to persist malicious behavior across deployments. AML.T0017 Develop Capabilities — Resource Development (AML.TA0003): Adversary developing poisoned components specifically for supply chain insertion. |
